# Spec2 Review - 公共比赛浏览模块

## 评审摘要
Spec2（公共比赛浏览模块）已实现公共比赛列表、比赛详情、球队详情和球员详情功能。整体实现良好，符合spec的核心要求，但存在一些性能问题和细节需要完善。

## 规范要求与实现检查

### 1. 公共比赛列表浏览 ✓ 基本符合

#### 要求：提供公共比赛列表端点和页面
- **实现检查**：`GET /matches` 端点已实现（match_handler.go:20），前端页面 `/matches` 已实现（matches/page.tsx）✓
- **筛选支持**：支持 `competition`、`season`、`page`、`pageSize` 查询参数 ✓
- **分页格式**：返回 `items`、`page`、`pageSize`、`total` 字段 ✓
- **空状态处理**：当 `items.length === 0` 时渲染空状态 ✓

### 2. 公共比赛详情浏览 ✓ 符合

#### 要求：提供公共比赛详情端点和页面
- **实现检查**：`GET /matches/:id` 端点已实现（match_handler.go:46），前端页面 `/matches/[matchId]` 已实现 ✓
- **基础信息**：返回比赛基本信息（主客队、比分、时间、赛事等）✓
- **404处理**：当比赛不存在时返回 `404 NOT_FOUND` ✓

#### 问题：
- 无

### 3. 比赛详情公共聚合数据 ✓ 符合但有性能问题

#### 要求：包含评分平均值、球员评分排行榜、最近评论
- **实现检查**：
  - `GetMatchAggregates()` 计算比赛、主队、客队平均评分和打卡人数 ✓
  - `GetPlayerRatingSummary()` 返回球员评分排行榜（前10名）✓
  - `GetRecentReviews()` 返回最近10条评论 ✓
- **小数规则**：使用 `ROUND(AVG(...)::numeric, 1)` 保留一位小数 ✓
- **空数据处理**：当无数据时返回 `null` 或空数组 ✓

#### 问题：
- **N+1查询**：`GetRecentReviews()` 中对每个评论执行单独的查询加载标签（第187-200行）。应使用 `Preload("Tags")` 一次性加载所有标签。
- **无样本提示**：前端显示 "No samples"，符合spec要求。

## 技术问题

### 高优先级
1. **N+1查询性能问题** - `match_repository.go` 中 `GetRecentReviews()` 方法对每个评论执行单独的查询加载标签，应优化为批量加载。

### 中优先级
2. **DTO完整性检查** - 需要确认所有响应字段与spec要求一致。当前DTO结构看起来完整，但需要验证与spec中"needed for display"要求的匹配度。
3. **边界情况测试** - 需要测试极端情况：超大pageSize、无效ID、不存在资源等。

### 低优先级
4. **API版本标识** - API根路径返回版本信息，但未在端点路径中体现版本前缀。
5. **错误响应一致性** - 检查所有错误响应是否遵循统一的错误格式。
