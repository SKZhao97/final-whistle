# Final Whistle `/me` 页档案感方向草案

## 1. 目标

如果 Final Whistle 走 A-first 路线，`/me` 页不应只是“个人中心”。

它更应该是：

> 我的足球观赛档案

这份文档用来说明 `/me` 页未来应如何从“统计 + 列表”走向“档案感”。

---

## 2. `/me` 页的角色

它不应该主要承担：

- 账户设置
- 通用个人中心
- 杂项管理入口

它应该主要承担：

- 我的赛后记录沉淀
- 我的赛季轨迹
- 我的偏好与判断风格
- 我的足球记忆回看

---

## 3. 档案感的定义

所谓“档案感”，不是简单把记录列出来。

它需要让用户看到：

- 我记录过什么
- 我重复关注什么
- 我这个赛季一路是怎么走过来的
- 哪些比赛对我更重要

也就是说：

```text
list of data
≠
personal archive
```

---

## 4. `/me` 页应有的四层结构

```text
┌──────────────────────────────┐
│ 1. Identity                  │
│ 我是谁，我正在记录什么        │
└──────────────────────────────┘
               ↓
┌──────────────────────────────┐
│ 2. My Patterns               │
│ 我的记录习惯与偏好            │
└──────────────────────────────┘
               ↓
┌──────────────────────────────┐
│ 3. My Archive                │
│ 我的比赛记录时间线            │
└──────────────────────────────┘
               ↓
┌──────────────────────────────┐
│ 4. Highlights / Memory       │
│ 值得回看的高分记录与关键比赛   │
└──────────────────────────────┘
```

---

## 5. Layer 1: Identity

这一层不是用户头像区，而是整个档案页的开场。

建议表达：

- 我的名字
- 一句关于“我的 Final Whistle” 的身份描述
- 当前累计记录数

这里应该有一种感觉：

```text
This is my football memory archive.
```

---

## 6. Layer 2: My Patterns

这是 `/me` 页最有潜力做出差异化的地方。

当前已有的一些字段：

- checkInCount
- avgMatchRating
- favoriteTeam
- mostUsedTag
- recentCheckInCount

未来可以继续往“模式”而不是“裸指标”方向表达。

例如：

- 我最常记录的球队
- 我最常使用的情绪标签
- 我给分偏高还是偏苛刻
- 我最近最常记录哪类比赛

这层的目标不是统计炫技，而是帮助用户看到：

> 原来我是这样看球的。

---

## 7. Layer 3: My Archive

这层是档案页的正文。

当前的 history list 已经是雏形，但未来要更强调“回看价值”。

要点：

- 每一条历史记录都要像一张可回看的赛后条目
- 比赛上下文要清晰
- 自己当时的判断要突出
- 标签和短评要能快速唤起记忆

重点不是把信息塞满，而是：

- 一眼回想起那场比赛
- 一眼看到自己当时怎么判断

---

## 8. Layer 4: Highlights / Memory

这是让 `/me` 真正有档案感的关键增强层。

它不一定要第一阶段就实现，但方向上值得保留。

可以想象的模块包括：

- 我给过最高分的比赛
- 我最常回看的记录
- 我最常打标签的比赛类型
- 本赛季的“经典记录”

这会让 `/me` 从“记录列表页”升级成“我的足球记忆陈列页”。

---

## 9. 视觉语言建议

`/me` 页应延续比赛详情页的语言：

- 记录册感
- 编辑感
- 克制留白
- 有归档感

而不是做成常见 dashboard 风格。

不建议：

- 卡片过多
- 指标宫格太重
- 看起来像后台数据面板

更推荐：

- 一页档案
- 几个真正有意义的模块
- 文本和记录本身成为主角

---

## 10. `/me` 页成功的标准

用户来到 `/me` 时，不该只感觉：

- 我有多少条数据

而应该感觉：

- 我真的在积累自己的足球观赛史

如果用户开始想回看旧记录、比较不同比赛、把这里当作自己的足球记忆页，那 `/me` 才真正成立。
